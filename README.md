# AI PDF Invoice Extractor API

FastAPI backend service that extracts structured invoice data from raw text and PDF files using OpenAI LLM models. Returns clean structured JSON with invoice_number, date, company and total_amount.

## Why This Project

Manual invoice processing is slow, error-prone, and expensive to scale. This API automates the extraction of key invoice fields using large language models - turning unstructured documents into structured, machine-readable data in seconds.

Real-world applications:

- **Invoice automation** - eliminate manual data entry from invoices
- **Document processing pipelines** - integrate into existing OCR and document workflows
- **Accounting integrations** - feed structured invoice data directly into ERP or accounting systems
- **ETL pipelines** - extract, transform and load invoice data at scale

## How It Works

```
PDF / Raw Text
      |
      v
PDF text extraction (pdfplumber)
      |
      v
OpenAI LLM field extraction
      |
      v
Structured JSON Response
```

Upload a PDF or send raw text - the API extracts text, sends it to OpenAI for field recognition, and returns a validated JSON object with the key invoice fields.

## Features

- **Dual input support** - accepts both raw text and PDF file uploads
- **LLM-powered extraction** - uses OpenAI models for intelligent field recognition
- **PDF text extraction** - extracts text from all pages using pdfplumber
- **Structured output** - returns validated JSON with company, invoice number, date and total
- **Async pipeline** - non-blocking HTTP calls with httpx for high throughput
- **Input validation** - Pydantic schemas for request and response validation
- **Error handling** - meaningful HTTP error codes for invalid files, empty PDFs and API failures
- **Production-style architecture** - clean separation of concerns, dependency injection and structured error handling
- **Auto-generated docs** - interactive Swagger UI at `/docs`

## Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/andrei-ameliugin/ai-pdf-invoice-extractor.git
cd ai-pdf-invoice-extractor

# 2. Create a virtual environment
python -m venv .venv && source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Configure your API key
echo "OPENAI_API_KEY=sk-..." > .env

# 5. Run the server
uvicorn main:app --reload
```

The API docs are available at **http://127.0.0.1:8000/docs**.

## API Endpoints

### `POST /extract`

Extract structured data from raw text.

**Content-Type:** `application/json`

```bash
curl -s -X POST http://127.0.0.1:8000/extract \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Invoice #104122\nDate: 2026-03-02\nFrom: Flowers Corp\nTotal: $1,250.00"
  }' | python -m json.tool
```

### `POST /extract-pdf`

Extract structured data from a PDF invoice.

**Content-Type:** `multipart/form-data`

```bash
curl -s -X POST http://127.0.0.1:8000/extract-pdf \
  -F "file=@invoice.pdf" | python -m json.tool
```

The PDF endpoint reuses the same extraction logic as `/extract` - no duplicated OpenAI calls.

### Example Response

```json
{
  "data": {
    "company": "Flowers Corp",
    "invoice_number": "104122",
    "date": "2026-03-02",
    "total_amount": "1250.00"
  },
  "model_used": "gpt-4.1-mini",
  "usage_tokens": 105
}
```

## Error Handling

| Scenario | Status Code | Detail |
|---|---|---|
| Non-PDF file uploaded | 400 | Expected a PDF file |
| Empty file | 400 | Uploaded file is empty |
| Corrupt or invalid PDF | 400 | The uploaded file is not a valid PDF |
| No extractable text in PDF | 400 | The PDF contains no extractable text |
| Empty or missing text field | 422 | Validation error |
| OpenAI API failure | 502 | Upstream API failure |

## Configuration

| Variable | Required | Default | Description |
|---|---|---|---|
| `OPENAI_API_KEY` | Yes | - | Your OpenAI API key |
| `OPENAI_MODEL` | No | `gpt-4.1-mini` | OpenAI model to use |
| `OPENAI_API_URL` | No | `https://api.openai.com/v1/responses` | OpenAI API endpoint |
| `OPENAI_TIMEOUT` | No | `30.0` | Request timeout in seconds |
| `LOG_LEVEL` | No | `INFO` | Logging level |
